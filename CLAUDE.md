# Ledger — working agreements

## Pull requests
- **Every change ships as its own new pull request against `main`.** Do not commit
  follow-up work onto a branch whose PR has already merged, and do not reuse a merged
  PR. Start a fresh branch off the latest `main` for each new piece of work and open a
  new PR for it.
- **Each new request/prompt gets its own fresh branch and its own PR.** Never stack a
  new request's work onto a previous request's branch, even if that branch's PR is still
  open. Branch from the latest `main` every time.
